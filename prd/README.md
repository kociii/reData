# 智能表格数据提取系统 - 文档索引

## 📚 文档概览

本目录包含智能表格数据提取系统的完整产品文档，涵盖需求分析、界面设计、开发计划和技术实现等各个方面。

## 📄 文档列表

### 1. [prd.md](./prd.md) - 产品需求文档
**内容概要**: 详细描述产品的业务背景、核心功能需求、业务流程和非功能性需求。

**主要章节**:
- **项目概述**: 业务痛点、AI 解决方案优势、项目价值
- **核心功能需求**: 数据提取、存储、AI 配置、文件导入、数据展示
- **业务流程**: 主流程图、表头识别流程、数据提取流程
- **技术栈**: 前端、桌面框架、后端、AI 集成
- **处理规则**: 表头识别、数据提取、空行处理、多 Sheet 处理
- **非功能性需求**: 性能、可用性、可靠性、安全性、扩展性
- **用户界面设计要求**: 主要页面功能描述
- **风险与挑战**: 技术风险和业务风险分析
- **项目里程碑**: 4 个阶段的开发计划
- **成功指标**: 准确率、稳定性、用户满意度

**适用人群**: 产品经理、项目经理、业务人员、开发团队

### 2. [design.md](./design.md) - 界面设计文档
**内容概要**: 使用 ASCII 图形详细描述系统的界面布局、交互流程和视觉设计。

**主要章节**:
- **整体布局**: 应用主框架结构
- **处理界面**: 文件列表、处理详情、Sheet 预览、提取结果展示
- **结果页面**: 数据表格、搜索筛选、编辑对话框
- **设置页面**: AI 配置管理、配置对话框
- **数据流程图**: 文件处理流程可视化
- **组件交互关系**: 前端组件树结构
- **状态管理**: Pinia Store 架构设计
- **关键交互流程**: 开始处理、实时进度更新
- **响应式设计**: 窗口尺寸适配方案
- **主题和样式**: 颜色方案、字体规范
- **错误处理和提示**: 错误提示样式、成功提示样式
- **快捷键设计**: 全局快捷键、页面快捷键

**适用人群**: UI/UX 设计师、前端开发工程师、产品经理

### 3. [plan.md](./plan.md) - 开发实施计划
**内容概要**: 详细的项目实施计划，包括技术架构、开发步骤、验证方法和风险管理。

**主要章节**:
- **Context**: 项目背景和核心需求总结
- **技术架构设计**: 项目结构、核心模块设计
- **数据库设计**: 完整的表结构和索引设计
- **核心处理流程**: 文件导入、处理流程、实时进度更新
- **AI Prompt 设计**: 表头识别和数据提取的 Prompt 模板
- **实施步骤**: 9 个阶段的详细开发计划（15 天）
  - Phase 1: 项目初始化
  - Phase 2: 数据库和基础服务
  - Phase 3: AI 集成和 Excel 解析
  - Phase 4: Tauri Commands
  - Phase 5: 前端 - 配置管理
  - Phase 6: 前端 - 处理界面
  - Phase 7: 前端 - 结果页面
  - Phase 8: 测试和优化
  - Phase 9: 打包和部署
- **关键技术点**: 并行处理、去重、进度更新、暂停/恢复
- **验证方法**: 功能验证、性能验证、错误处理验证
- **关键文件路径**: 前端和后端关键文件列表
- **风险和注意事项**: AI 成本、准确率、性能、数据安全

**适用人群**: 项目经理、技术负责人、开发团队

---

### 4. [dev.md](./dev.md) - 技术文档
**内容概要**: 详细的技术栈选型、数据结构设计、核心模块实现和技术方案。

**主要章节**:
- **技术栈**:
  - 前端: Vue 3 + TypeScript + Vite + Pinia + Element Plus
  - 桌面框架: Tauri 2.x
  - 后端: Rust + SQLite + OpenAI SDK
  - 核心依赖库详细列表
- **数据库设计**:
  - 数据库选型理由
  - 4 张表的完整结构和字段说明
  - 索引设计
- **项目结构**: 完整的目录树和文件说明
- **核心模块设计**:
  - 前端: 路由、状态管理（Pinia Stores）
  - 后端: Tauri Commands、Services
- **关键技术实现**:
  - 并行处理（tokio 异步）
  - 手机号去重（UNIQUE 约束）
  - 实时进度更新（Tauri 事件系统）
  - 暂停/恢复（Arc<Mutex<bool>>）
- **AI Prompt 设计**: 表头识别和数据提取的完整 Prompt
- **性能优化策略**: 数据库、前端、后端优化方案
- **安全性设计**: API Key 加密、数据隔离、输入验证
- **错误处理策略**: 错误分类和处理机制
- **测试策略**: 单元测试、集成测试、端到端测试

**适用人群**: 开发工程师、架构师、技术负责人
