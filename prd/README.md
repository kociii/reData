### 4. [dev.md](./dev.md) - 技术文档（How - Tech）

**职责范围**：回答"技术如何实现"的问题
- ✅ 详细的技术栈选型和理由
- ✅ 核心依赖库和版本
- ✅ 项目结构（完整的目录树）
- ✅ 核心模块实现（代码级别的设计）
- ✅ 关键技术实现（具体的技术方案）
- ✅ 性能优化策略
- ✅ 安全性设计
- ✅ 错误处理策略
- ✅ 测试策略
- ❌ 不包含：业务需求、界面设计、开发步骤、项目管理

**主要章节**:
- 技术栈：
  - 前端: Nuxt 3.18+ + TypeScript + Nuxt UI 3.x + Pinia
  - 桌面框架: Tauri 2.x
  - 后端: Rust + SQLite + OpenAI SDK
  - 核心依赖库详细列表（calamine、rusqlite、reqwest、tokio 等）
- 数据库设计：
  - 数据库选型理由（SQLite 的优势）
  - 表结构详细说明（字段类型、约束、外键）
  - 索引设计和性能考虑
  - JSON 字段存储方案
- 项目结构：完整的目录树和文件说明
- 核心模块设计：
  - 前端: 路由设计、Pinia Stores（processing、result、config、project）
  - 后端: Tauri Commands（project、field、file、processing、config、result）、Services（excel_parser、ai_client、extractor、storage）
- 关键技术实现：
  - 并行处理（tokio 异步运行时）
  - 动态字段存储（JSON 字段 + 动态索引）
  - 灵活去重（单字段/多字段组合去重）
  - 实时进度更新（Tauri 事件系统）
  - 暂停/恢复（Arc<Mutex<bool>>）
  - AI Prompt 动态生成（基于项目字段定义）
- 性能优化策略：数据库优化、前端优化、后端优化
- 安全性设计：API Key 加密、数据隔离、输入验证
- 错误处理策略：错误分类和处理机制
- 测试策略：单元测试、集成测试、端到端测试

**适用人群**: 开发工程师、架构师、技术负责人
